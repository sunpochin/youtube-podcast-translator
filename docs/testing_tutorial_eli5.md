# 🧸 小朋友也能懂的單元測試 (Unit Test) 與 Mock (模擬替身) 教學

嗨！如果你覺得「單元測試」和「Mock」聽起來像外星話，別擔心！我們用最簡單的**玩具城堡與特技替身**故事，讓你一分鐘搞懂它們！

---

## 🧱 1. 什麼是單元測試 (Unit Test)？

想像一下，你正在用樂高積木蓋一座巨大的「太空城堡」（這就是你的**專案程式**）。

*   **大問題**：如果城堡蓋好後，你按下開關，城堡卻沒有發光也沒有動，你該怎麼知道是哪一塊積木壞掉了？是電池？馬達？還是發光的積木？你要把整座城堡拆掉檢查嗎？
*   **解決方案（單元測試）**：在把每一塊積木裝上去之前，你先單獨拿電池接上燈泡，看看它會不會亮。這種**「單獨測試最小積木（函數/單一功能）」**的動作，就叫做**單元測試**。

### 程式碼範例：
我們有一個幫字串加上愛心的功能：
```javascript
// 這是我們的積木 (函數)
export function addHeart(name) {
  return `❤️ ${name} ❤️`;
}
```
我們的**單元測試**就像在旁邊做實驗：
```javascript
// 實驗：如果我丟進 "小乖"，出來的會是 "❤️ 小乖 ❤️" 嗎？
const result = addHeart("小乖");
assert.strictEqual(result, "❤️ 小乖 ❤️"); // 答對了！測試通過！
```

## 🎭 2. 什麼是 Mock (模擬替身)？

現在，你的積木變得更複雜了。你設計了一塊「火箭發射按鈕」積木。
當你按下按鈕時，它會做三件事：
1.  打電話給太空總署（聯絡真實的 Gemini/Ollama AI 伺服器，需要網路，還要收錢）。
2.  點燃真的火箭引擎（執行 GitOps 指令，會改變你真實的 GitBook 筆記）。
3.  發射火箭。

*   **大問題**：你只是想測試這個「按鈕按下去，紅燈會不會亮」而已，難道你每次測試，都要真的花好幾萬塊發射一枚火箭、還要把真的太空港炸掉嗎？而且如果今天外面下大雨、網路斷線（伺服器當機），你的測試不就直接失敗了？
*   **解決方案（Mock）**：我們在測試時，把「真的太空港」和「真的火箭」藏起來，換成一個**「玩具太空港（Mock）」**和**「特技替身（Stub）」**。
    *   這個玩具太空港長得跟真的一模一樣，按鈕按下去時，它不會真的發射火箭，只會乖乖舉起一張牌子說：「收到！火箭發射成功囉！」
    *   這種**「用假的玩具代替真的危險/昂貴服務」**的技巧，就叫做 **Mock（模擬）**。

## 🔬 3. 在我們專案中，我們 Mock 了什麼？

在我們的 `youtube-podcast-translator` 中，有兩個非常危險且昂貴的「真實火箭」：
1.  **AI 翻譯 (Gemini / Ollama)**：每次翻譯都要上網、花時間，Gemini 還可能要收費。
2.  **Gitops 推送 (GitBook)**：每次發佈都會真的去改你的 Git 紀錄、推送到雲端。

為了安全且快速地測試，我們在 [test/slug.test.js](file:///Users/pac/codes/interview/youtube-podcast-translator/test/slug.test.js) 裡放了「特技替身」：

### A. 模擬 AI 客戶端 (Mocking Fetch/Gemini)
我們告訴測試系統：「等一下程式如果打電話給 AI 伺服器，請攔截它，直接給它寫好的假翻譯，不要真的上網！」
```javascript
// 攔截網路 fetch，當程式想找 Ollama 時：
t.mock.method(globalThis, 'fetch', async () => {
  return {
    ok: true,
    json: async () => ({
      message: { content: '這是假的翻譯（替身出來打招呼）' }
    })
  };
});
```

### B. 模擬 Git 司令官 (Mocking Git Commands)
我們告訴測試系統：「程式如果想在電腦執行 `git commit` 或 `git push`，請攔截它，不要真的去推程式碼，只要在小本本上記錄程式有呼叫這個指令就好！」
```javascript
// 攔截 Git 指令執行器：
const gitCalls = [];
t.mock.method(gitExecutor, 'exec', async (cmd, args) => {
  gitCalls.push({ cmd, args }); // 記錄下來，證明程式有想發射火箭
  return { stdout: 'Mocked git output' }; // 假裝成功執行
});
```

## 🏆 總結：為什麼面試官超愛這個？

當你在面試時，你可以自豪地說：
> 「我的專案實作了完整的單元與整合測試。為了保證測試的**速度**、**穩定性**與**安全性**，我對 AI 服務與 GitOps 指令進行了 **Mock 隔離**。這讓測試完全不依賴網路與外部環境，在一秒內就能在沙盒中安全執行完畢，且覆蓋率達到了 61% 以上！」

這會讓面試官覺得你非常有商業專案開發的經驗，懂得保護系統資源與寫出高品質、可測試的代碼！
